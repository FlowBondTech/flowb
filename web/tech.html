Auth System

Zero-dependency HS256 JWT implementation (src/server/auth.ts). Supports Telegram initData HMAC validation, Farcaster SIWF via Neynar, and standard Bearer token auth. JWTs contain sub (user ID), platform, and platform-specific fields.

Multi-Platform Architecture

Six platform frontends, all backed by the same API. Identity resolution unifies users across platforms.

🤖

Telegram Bot

Grammy framework. Inline keyboards, mini app launcher, deep links. Primary interaction channel. Bot: @Flow_b_bot

Auth: initData HMAC → JWT
🟣

Farcaster Mini App

Next.js static export at fc.flowb.me. Farcaster SDK for frames and actions. SIWF authentication.

Auth: SIWF signature → Neynar verify → JWT
🌐

Web App

Static HTML/CSS/JS at flowb.me. Vanilla JavaScript, no framework. Farcaster native SIWF login.

Auth: SIWF or guest session
📱

Mobile App

Expo / React Native at me.flowb.app. EAS builds for iOS and Android. Shared API layer with web.

Auth: Supabase Auth + JWT
💬

WhatsApp

Cloud API integration. Natural language commands, event search, crew coordination via chat.

Auth: WhatsApp user ID → JWT
🔒

Signal

Signal REST API integration. Privacy-focused messaging with event and crew commands.

Auth: Signal user ID → JWT

Notification System

Six notification channels with priority-based routing, deduplication, and user preference controls.

📨

Telegram DM

Primary channel. Rich formatting with HTML. Inline keyboards for actions. Rate limited per user.

🔔

Farcaster Push

Farcaster notification API. Fallback when TG unavailable. Frame-based interactive notifications.

📧

Email (Resend)

Transactional emails: welcome, verification, digest, event reminders. FlowB branded templates.

📲

Expo Push

Native push for mobile app. Event reminders, crew updates, point milestones.

💬

WhatsApp

Cloud API messages. Template-based for marketing, free-form for user-initiated conversations.

🔒

Signal

Privacy-focused notifications via Signal REST API. Text-only, no tracking pixels.

Routing Logic

// Priority tiers: urgent // All channels simultaneously (security, account) important // Primary + fallback (crew updates, events) info // Primary channel only (points, streaks) // Channel priority (per user, based on most-active platform): // 1. Telegram DM (if TG user) // 2. Farcaster push (if FC user) // 3. Email (always available) // 4. Expo Push (if mobile app installed) // Dedup: flowb_notifications_sent table prevents double-sends

Developer Guide

Setup instructions, environment configuration, and development conventions.

Quick Start

# Clone and install git clone https://github.com/flowb-app/flowb.git cd flowb && npm install # Configure environment cp .env.example .env # Edit .env with your keys (see below) # Run development server npm run dev # Fastify server with hot reload npm run bot # Telegram bot (Grammy) npm run web # Web app (static server) # Mini apps cd miniapp/farcaster && npm run dev # Next.js at :3000 cd miniapp/telegram && npm run dev # Vite at :5173 # Mobile cd mobile && npx expo start # Expo dev server

Project Conventions

// File organization: src/server/ // Fastify routes and middleware src/telegram/ // Grammy bot handlers src/plugins/ // Modular domain plugins src/services/ // Standalone service modules src/utils/ // Shared utilities (supabase, logger, etc.) src/core/ // FlowBCore orchestrator web/ // Static web app (flowb.me) miniapp/ // Platform mini apps mobile/ // Expo mobile app supabase/ // DB migrations // Code style: // - TypeScript throughout // - 2-space indent, no semicolons (auto-formatted) // - fireAndForget() for non-blocking side effects // - sbQuery/sbInsert/sbPatch/sbUpsert for all DB ops // - alertAdmins() for real-time admin notifications
